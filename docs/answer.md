## 종합 보고서: AI 중심 소프트웨어 엔지니어링 (AISE)

**핵심 주제:** 본 보고서는 AI 중심 소프트웨어 엔지니어링 (AISE)의 새로운 접근 방식에 대해 분석합니다. 기존 소프트웨어 엔지니어링 방식의 한계를 극복하고, 대규모 언어 모델 (LLM)을 활용하여 개발 과정을 효율적이고 재현 가능하게 만드는 것을 목표로 합니다.

**결론:** AISE는 ‘Intent, State, Event, Trajectory, Replay’라는 핵심 요소를 기반으로, Markdown 문서를 활용한 Recursive Build Agent를 통해 자연어 명세로부터 실행 가능한 코드를 자동 생성하는 파이프라인을 구축합니다. 이러한 접근 방식을 통해 소프트웨어 개발의 투명성을 높이고, 개발 과정의 자동화 및 재현 가능성을 확보하여 궁극적으로 소프트웨어 개발 생산성을 향상시킬 수 있습니다.

**세부 내용:**

* **AISE의 핵심 개념:** AISE는 LLM을 핵심 개발 도구로 활용하고, ‘Intent, State, Event, Trajectory, Replay’라는 5가지 핵심 요소를 통해 개발 과정을 체계화하는 새로운 접근 방식입니다. 이는 의도, 상태, 이벤트, 궤적, 재현이라는 요소들을 명확히 정의함으로써 개발 과정의 구조화 및 자동화를 가능하게 합니다.
* **Markdown 기반의 Recursive Build Agent:** AISE의 핵심 구현체인 Recursive Build Agent는 Markdown 문서를 실행 가능한 사양으로 활용하고, AI와 인간 간의 의사소통을 위한 중간 표현 (IR)으로 활용합니다. Agent는 명세 파싱, 코드 합성, 검증 및 피드백, 그리고 역방향 기록의 4단계 루프를 통해 재귀적으로 시스템을 발전시키며, Markdown 문서와 생성된 코드를 완벽하게 동기화하여 투명성을 확보합니다.
* **MaDD 파이프라인:** AISE의 주요 방법론인 MaDD (Markdown-Driven Development) 파이프라인은 프로젝트의 모든 구성 요소를 Markdown 문서로 관리하고, Build Agent를 통해 자동으로 코드를 생성 및 수정합니다. 이는 자연어 명세로부터 실행 가능한 코드를 자동으로 생성하는 파이프라인 구축에 초점을 맞추고 있습니다.

**향후 연구 방향:**

* 다양한 LLM 모델과의 통합 및 성능 최적화
* 개발 과정의 자동화 및 재현 가능성 향상을 위한 방법론 연구
